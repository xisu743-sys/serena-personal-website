01

手动刷的那段时间

为了找 YouTube 频道定位，我空下来就刷视频，每天两三个小时。 现有工具要么要付费，要么数据只在侧边栏被动显示， 最想捕捉的「低粉异常爆款」根本没有办法系统化发现。

🔍

看不到增量

插件只显示发布时间，无法直接看到播放量的实时增量，只能靠发布时间+现有播放量大概猜测。

💸

付费门槛高

vidIQ、ViewStats 数据展示在侧边栏，信息零散，且前期不愿投入成本去订阅。

⏰

没有主动提醒

所有工具都要我主动去看，没有一个能在爆款出现时第一时间通知我。

📡

找不到低粉爆款

最有参考价值的是 5k–10k 小频道的异常增长，但这类视频完全淹没在大频道内容里。

03

真正卡住的三个地方

01 增量计算

滚动窗口，不是累计

从发布时间起算的播放量是固定值，没法判断最近有没有爆发。 换成以 Date.now() 倒推才对—— 这是整个项目花时间最多的一步。

// ✅ 正确：滚动倒推
cutoff = Date.now() - 24h
delta = views_now - views_at(cutoff)
02 定时任务

第一天没动静，原因在 endpoint

QStash 接入后首日静默，无报错。定位到是 Upstash regional URL 配置错误， 改掉之后定时同步才正常运行。

→ 00fe7aa

03 API 配额

两处改动，配额从 100 降到 1

YouTube API 按调用单位计费。频道搜索换 forHandle， 视频列表换 playlistItems， 各省 99 单位。

第一次收到爆款通知时，我知道这个工具的闭环成立了——
这次跑在生产环境里的，不是 demo。

规则触发 · 企业微信推送 · 20 个视频命中

每个功能，背后都有一个具体的发现

MVP 跑通之后，每加一个功能都是因为用着用着发现缺了什么。

↳ 通知了但不知道为什么触发

从单一阈值升级成自定义条件组合，任意/全部匹配都支持，命中后清楚告诉你触发了哪条规则。

24h 增量 条件组合 企业微信·飞书·邮箱

↳ 爆款看完就忘，没地方沉淀

把爆款视频归入"动画科普""讽刺类"等样本组。原来只是监控，现在能把好内容存下来，形成可复盘的内容库。
